# Erjia Schedule Converter

## 用途

當使用者提供兒家班表 PDF，要求：
- 分析排班脈絡
- 重建成 Excel
- 復刻顏色與儲存格內容
- 推理班表規則

此 skill 專門負責把兒家班表 PDF 還原成可編輯的 `.xlsx`，並整理可重用的排班解讀規則。

## 目標定義（導入版）

此 skill 不只做「轉檔」，還要支援「人資系統導入前的規格對齊」：

- 先還原班表真實內容（顏色、時段、多段班、場域）
- 再比對人資系統既有班別設定
- 找出缺漏班別、命名不一致與工時定義衝突
- 產出可匯入資料前的班別字典與 mapping 規則

## 核心原則

- `pdfplumber` 是主工具，優先用 `extract_table()` 取整張表。
- `pdftotext -layout` 只能當輔助比對，不可當最終真值。
- 若 PDF 內有多行儲存格，必須保留換行，不可把 `女樓08-16` 拆成 `女` 與 `樓08-16`。
- 若使用者已明確指定顏色語意，使用者規則優先於顏色推測。
- 預設輸出新檔，不直接覆寫原始 Excel 或 PDF。

## 已確認的班表語意

### 顏色規則

- 白底：上班日。儲存格文字就是實際上班時段或值勤內容。
- 紅底：休息日或例假日。目前先統一視為非上班日。
- 藍底：特休假。

### 工時字串規則

- `08-13`：`08:00-13:00`，5 小時。
- `15-23`：`15:00-23:00`，8 小時。
- `13-23`：`13:00-23:00`，10 小時。
- `08-13/18-23`：多段班；上午 5 小時 + 晚上 5 小時。
- `女樓13-23`、`男三15-23`：表示工作場域 + 時段。
- `子琳08-13`：工讀補位型短班，不一定遵循正職輪值節奏。
- `子琳特教伴讀\n08-13男樓\n13-15女樓`：同一日多段且跨場域，必須保留換行或至少保留完整內容。

### 補充判讀

- 單格中的 `夜班` 表示值大夜或夜間輪值，不等於一般 8 小時白班。
- 單格中的 `事假`、`特休`、`颱風假` 等文字，優先視為假別或停班狀態，不再解讀為工時。
- 單格只出現 `2`、`3`、`4` 這類數字時，通常是原表內部標記或補位註記，先原樣保留，不能自行改寫成工時。
- `白底` 不是單一班別，而是「有出勤」的集合；實際班別要看下方時段說明列。
- 判讀順序固定為：先看上層顏色/出勤，再看下層文字時段（這是保育生輔股的必要規則）。
- 若白底格為空白且下方無清楚標註，視為 `未知班別`，不得自動當成 `08-18` 或 `15-23`。
- 若上方「人員列」某日為空白，必須往下看同欄的崗位列（如 `男二`、`男三`、`男樓夜班`、`女二`、`女三`、`女樓夜班`）；若該格含有同一姓名，需把該人對應時段回填到上方人員列。
- 回填時若下方格是 `婷予08-13/18-23` 這種格式，應回填 `08-13/18-23`；若是 `亮寧/婷予` 這種多人且未寫時段，才回填崗位名稱本身。
- 這種回填屬於「推理補全」，建議以黃色或其他明顯方式標記，避免與原始表手填內容混淆。

## 重建流程

### 1. 抽表

優先：

```python
import pdfplumber

with pdfplumber.open(pdf_path) as pdf:
    table = pdf.pages[0].extract_table()
```

處理方式：
- 第 1 列通常是日期欄。
- 第 2 列通常是星期欄。
- 兩列一起決定欄位結構。
- 空白標題欄要過濾掉，只保留真正的日期與統計欄。

### 2. 決定資料區範圍

常見主資料區：
- 人員列：如 `偉培`、`郁茵`、`卉蓁`、`婷予`、`亮寧`
- 補位／工讀列：如 `工讀\n子琳`

常見後續區塊：
- `原則`
- `行政班`
- `男二`
- `男三`
- `男樓夜班`
- `女三`
- `女樓夜班`
- `行事曆`

規則：
- 若使用者要「主班表」，先輸出人員主表。
- 若使用者要「完整復刻」，上述後續區塊也要保留，可放同一張工作表下半段或分工作表。

### 3. 寫回 Excel

建議使用 `openpyxl` 或 `pandas + openpyxl engine`：

- 開啟 `wrap_text=True`
- 文字水平、垂直置中
- 保留多行內容中的 `\n`
- 日期欄依原順序展開
- 右側統計欄保留：`大夜班數`、`休假`、`特休`、`其他`

### 4. 顏色復刻

先從 PDF 的矩形填色取得候選色：

```python
page.rects
rect["non_stroking_color"]
```

但套色時，不要只依 RGB 亂猜，必須先套用已知語意：
- 白底 -> 不填色或白色填滿
- 紅底 -> 休息日/例假日
- 藍底 -> 特休

其餘顏色若出現在 `行事曆` 或註記區，可先保留原色作視覺復刻，但不要自行延伸語意。

## 班別導入比對流程（新增）

### A. 先做班別盤點（從班表真值出發）

- 從班表抽出所有班別字串與變體，例如：
- `15-23`、`13-23`、`08-18`、`08-13/18-23`
- `女二午晚`、`女三午晚`、`男樓15-23` 等「場域+班別」混合寫法
- `空白白底`（需人工確認）與 `夜班`（固定定義）

### B. 再做系統班別設定比對

- 對照人資系統 `出勤管理 -> 班別設定` 是否已有對應班別碼
- 若班表有、系統沒有：列為 `缺碼`
- 若系統有、班表未用：列為 `未使用班別`
- 若名稱相似但時段不同：列為 `命名衝突`

### C. 導入前必做確認

- 一天是否只能選一種班別（若是，需先定義多段班拆解策略）
- `13-23` 是否視為標準班、延長工時或需拆段
- `寒暑假班` 與 `平日班` 是否共用代碼
- `空白白底` 的預設班別是否可被正式定義

### D. 落地節奏（兩階段）

- 第一步：列出「所有可能班別」與「缺漏代碼」，先補齊班別設定。
- 第二步：做 `Excel -> 系統匯入格式` 轉換，再進入批次匯入。

## 匯入模板規格（`排班表匯入 (3).xls`）

### 模板結構

- 工作表 `Sheet0`：
- 第 1 列：`*員編`、`姓名`、`YYYYMMDD` 日期欄
- 第 2 列：星期欄（只做顯示）
- 後續每列：每位員工的每日匯入值
- 工作表 `說明`：定義匯入語法與特殊符號（必須遵守）

### 每日儲存格匯入語法

- 只匯入班別：`班別代碼`
- 匯入班別 + 崗位 + 部門 + 地點：`班別代碼-崗位代碼,費用部門/地點`
- 國假類型：`國!勞動節`（`!` 用於國假描述）
- 假別：`#假別代碼`（例如 `#事`）

### 假別/休假代碼提醒

- `休息日 -> 休`
- `例假日 -> 例`
- `國假 -> 國`
- 請假一律 `#` 開頭，例如 `#事`、`#特`

## 班別代碼主檔（2026-03-29）

- 班別代碼主檔來源：使用者提供的「未來排班代碼參考」。
- 檔案位置：`Inbox/2026.03.28人資系統各單位班表導入/班別代碼參照_20260329.csv`
- 轉換時禁止使用 `狀態=停用` 的班別代碼（如 `b0000`、`b0029`）。

## 兒家班表轉匯入代碼的最低要求

- 必須先建立「班表字串 -> 班別代碼」mapping，不可直接以文字原樣匯入。
- `夜班` 預設對應 `b0002`（兒大夜，22:30-08:00）；若時段不一致需改列 `待確認`。
- 白底且有明確時段時，優先以時段比對代碼；白底空白一律列 `待確認`。
- 遇到多段班（例如 `08-13/18-23`）：
- 若系統允許以單一代碼表示，映射到對應代碼。
- 若系統只能單班別，先輸出 `待人工拆解`，不得硬塞單一班別。

## 已踩過的坑

- 用 `pdftotext -layout` 切欄時，`女樓08-16` 容易被錯拆成跨兩欄內容。
- 用固定字元寬切欄時，多行儲存格會造成欄位位移。
- 有些 PDF 的顏色矩形是背景塊，不一定與文字框完全重合，不能只靠單一字的座標配色。
- `pdfplumber.extract_table()` 雖然不完美，但對保留合併儲存格與換行內容，比純文字切欄穩定得多。
- 班表自由文字未標準化（如 `女二午晚`、括號註記、空白白底）會讓自動 mapping 失敗。
- 直接假設白底=固定班別，會導致匯入後工時與實際排班不一致。
- 若不先定義「未知班別」類別，會把模糊欄位誤塞到錯誤班別碼。

## 產出標準

- 至少產出一份可開啟的 `.xlsx`
- 日期欄完整
- 人員列完整
- 多行班別不被拆壞
- 白/紅/藍底語意正確
- 若無法 100% 確認某格，保留原文，不要擅自改寫

## 回覆使用者時應說明

- 哪些規則是從 PDF 推理
- 哪些規則是使用者明確指定
- 哪些欄位已正確還原
- 哪些欄位仍屬不確定、需人工確認
- 哪些班別在人資系統 `班別設定` 已存在、哪些仍缺漏
- 匯入前必須先確認的制度項目（例假/休息日、工時口徑、多段班拆解）
