# 排班作業

## 已驗證頁面

- 這個租戶的網址：`https://erp3.aoacloud.com.tw/HR/HRHB007S00.aspx`
- 核心 selector：
  - 員工選擇器：`#WorkScheduleOrgCtrl_Emp`
  - 週期選擇器 model：`vm.query.GroupByCycle`
  - 存檔按鈕：`#btnSave`

## 已驗證行為

- 對於使用 4 週彈性排班的員工，請在週期檢視中作業，不要只看一般月檢視。
- 2026 年 3 月案例已驗證的週期是 `n472`，涵蓋 `2026/03/12~2026/04/08`。
- 週期排班的存檔會先呼叫 `api/WorkScheduleMulti/Save`，再呼叫 `api/HRHB007S00/SaveWorkCalendar`。
- 日模式多段班的存檔會呼叫 `api/WorkScheduleMulti/Save`。

## 週期排班模式

- 先把員工載入到格子中。
- 將 `vm.query.GroupByCycle` 切到目標週期，再呼叫 `_GetTableRow()`。
- 在 `vm.controller.HoliDayClass = true` 時，使用 `_SetClass(...)` 套用工作日。
- 在 `vm.controller.HoliDayClass = false` 時，使用 `_SetClass(...)` 套用休假日。
- 存檔後重新整理，並驗證已保存的 `DaysInfo` 狀態。

## 月格批次模式

- 月排班時，不需要每點一格就存一次。
- 可以先把多位員工載入到同一個週期格中，再批次點格，最後一次存檔。
- 比較有效率的操作流程是：
  - 載入目標月份與週期
  - 先加進整批要修改的員工
  - 在上方 `班別` 選擇器選一個班別代碼
  - 點選所有要套用該班別的目標格子
  - 將 `班別` 選擇器切換到下一個班別代碼
  - 點選下一批目標格子
  - 重複直到所有班別群組都套用完
  - 最後只按一次右上角存檔
- 同一個頁面可以在最後存檔前混用不同班別代碼；重點是依班別代碼分批，不是依員工分批。
- 依班別代碼分組通常比依員工分組更快，因為每一批只需要改一次 selector。
- 最後存檔後仍要重新整理，並從頁面資料驗證，不要信任未存檔的格子狀態。
- 如果修正只影響少數員工，也可以把他們所有更動日期集中成每位員工一次存檔，或每個頁面一次存檔，只要最後都有重新整理並驗證即可。

已驗證範例：

- 員工：`劉春燕(302)`
- 週期：`n472`
- 範圍：`20260312` 到 `20260408`
- 平日：`b0027`（`晚餐膳務 12:00~20:30`）
- 星期六：`kd=4`（`休息日`）
- 星期日：`kd=0`（`例假`）

## ba000 多段班模式

- 不要試圖把多段時段直接寫進單一文字欄位。
- 先在週期/月模式把目標日期轉成 `ba000`。
- 再呼叫 `GoToDayMode()`。
- 在日模式中，編輯 `vm.Day.Rows` 裡對應的列。
- 針對目標日期列，更新：
  - `DutyClass = 'ba000'`
  - `WorkRanges = [...]`
  - `HoursSum`
  - `is_auto_multi = false`
- 日模式的列查找應使用 `Name` 或 `Empno`；那裡沒有 `EmpnoName`。

已驗證範例：

- 日期：`2026/03/25`
- 時段：
  - `08:00-12:00`
  - `18:00-23:00`

## 常見陷阱

- 在檢查法定休息日的月驗證規則裡，`特休` 不算 `例假`。
- `GoToDayMode()` 只會載入目前週期/月列中已經是 `ba000` 的日期。
- 如果你要做單日 `ba000` 編輯，要先把那一天轉成 `ba000`，再進入日模式。
