# Annual Plan Consistency Manager 年度計畫一致性管理員

## 🎯 核心任務
Gary 的年度計畫分佈在多個檔案中，為了避免「改了這本沒改那本」造成的資訊斷層，本技能定義了檔案間的連動規則與同步清單。

## 📂 關聯檔案清單 (Core Sync Group)
當以下檔案涉及「目標名稱」、「項目編號」、「關鍵 DDL」、「核心策略」的修改時，必須執行**全域搜尋並同步更新**：

1. **九宮格總表**: `2026_九宮格年度目標.md` (這是最高層級定義)
2. **策略書**: `2026_年度目標與意圖達成策略書.md` (包含意圖與 Doing 策略)
3. **甘特圖與分解**: `2026_項目甘特圖與倒推計畫.md` (包含進度與路線圖)
4. **專項計畫**: `2026_騎行計畫.md` (若涉及騎行目標調整)
5. **長遠願景**: `2031_五年後的願景信.md` (若涉及 2026 階段性成果描述的變更)

## 🔄 同步準則
- **目標名稱變更**: 若 4B 目標從「下屬工作日誌」改為「下屬工作追蹤」，必須更新所有檔案中的相關文字。
- **編號一致性**: 確保工作事業 A/B/C 在所有檔案中的代號（如 P1, P2）保持一致。
- **連動警告**: 若調整了甘特圖中的 DDL，應檢查九宮格或策略書中是否有對應的「時限」描述。

## 🛠 操作流程 (Agent Guidelines)
1. **識別**: 當使用者提出修改年度計畫的任何部分時，先識別該變動是否屬於「核心組件」。
2. **搜尋**: 使用 `grep_search` 在上述清單中搜尋舊字串。
3. **同步**: 使用 `multi_replace_file_content` 執行全域替換。
4. **驗證**: 檢查修改後的語境是否通順。
