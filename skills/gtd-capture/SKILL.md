# GTD 收集/捕捉 (Capture)

> 「你的頭腦是用來產生想法的，不是用來儲存想法的。」— David Allen

## 核心目的
協助使用者將腦中**所有未完成的事物**全部「倒出來」，達到「心如止水」的清空狀態。

## 使用時機
- 使用者說「我要整理一下」「大腦掃除」「Brain Dump」「收集」
- 定期（每週/每月）的 GTD Review
- 使用者感到焦慮、混亂、事情太多時

## 執行流程

### Step 1：選擇觸發清單
根據情境，使用以下兩份觸發清單引導使用者：

#### 📋 職業方面觸發清單
| 面向 | 觸發關鍵字 |
|------|-----------|
| **專案** | 已開始未完成的、需開始的、評估中的 |
| **對他人的承諾** | 老闆、同事、下屬、合作單位、客戶 |
| **需進行的溝通** | 電話、信件、LINE、電子郵件、會議記錄 |
| **待完成文件** | 報告、評鑑、提案、企畫、會議紀錄、進展匯報 |
| **財務** | 預算、應收應付、捐贈款、報銷 |
| **計畫/整理** | 短中長期目標、即將活動、年度行事曆、新系統 |
| **組織發展** | 架構圖、職權、新系統、文化、改革 |
| **管理** | 法務、保險、人事、政策/流程、訓練 |
| **人事** | 招聘、考核、升遷、薪資、員工發展 |
| **會議** | 近期議程、被要求參加的、需簡報的、需通知的 |
| **辦公室/場地** | 空間、設備、清潔、安全、維修 |
| **等待中** | 委派事項、回覆、決策、報銷、票據 |

#### 🧘 個人方面觸發清單
| 面向 | 觸發關鍵字 |
|------|-----------|
| **對他人的承諾** | 家人、朋友、歸還物品、債務 |
| **溝通** | 私人訊息、感謝、問候、社群貼文 |
| **近期活動** | 生日、紀念日、聚會、旅行、訪客 |
| **財務** | 帳單、投資、貸款、預算、保險、存錢目標 |
| **家庭/環境** | 維修、家電、收納、水電、清潔 |
| **健康** | 醫生、牙醫、體檢、飲食、運動 |
| **學習/成長** | 課程、書籍、技能、創意表達 |
| **交通** | 汽機車保養、維修、通勤 |
| **服裝** | 專業、休閒、運動、維護 |
| **休閒** | 想去的地方、攝影、運動、娛樂 |
| **社交** | 社區參與、人際經營 |
| **等待中** | 訂購品、維修中、借出物、他人回覆 |

### Step 2：逐項引導
- 每次只問一個面向：「在【專案】方面，你有什麼已開始但還沒完成的事嗎？」
- 不要催促，讓使用者自由聯想
- 使用者說「沒了」才進入下一個面向

### Step 3：即時記錄
- 每個項目立刻寫入 Notion（使用 `to_do` block）
- 分類歸到對應的面向標題下
- **不判斷**、**不評價**、**不排序** — 收集階段只管「倒出來」

### Step 4：確認完成
- 全部面向走完後，問：「還有什麼漏掉的嗎？」
- 確認後，將狀態從「收集中」改為「收集完成」
- 提醒使用者：下一步是「釐清 (Clarify)」— 逐項判斷每件事該怎麼處理

## 與其他技能的關係
- 收集完成後 → 使用 `gtd實踐` 技能進行釐清（要不要執行？2分鐘能完成？專案還是單步？）
- 行事曆項目 → 使用 `google-calendar` 技能加入行程
- 專案展開 → 使用 `discovery.md` 精神進行頭腦風暴

## 目標頁面
- **Notion 任務規劃頁：** `30d1fbf9-30df-81bb-9dc5-d6a2418dc5f3`
- **寫入方式：** Notion API `PATCH /blocks/{page_id}/children`
- **重要：收集階段的任務一律先放在「近一個月的任務規劃」頁面，不要直接寫進 Tasks DB。只有經過「釐清 (Clarify)」後確認要執行的，才移入 Tasks DB。**
