# Organization Operation Database Manager (組織運作資料庫管理技能)

## 🎯 核心使命
從 Gary 每天的實務工作與決策中提煉「智慧與經驗」，避免這些寶貴的經驗隨著時間消散。
這正是實踐 2031 願景信中「無損（Lossless）」境界的關鍵基礎：**讓每一次的經驗產出，都能 100% 轉化為系統的養分。**

## 📂 資料庫位置
*   **檔案路徑**：`/Users/leegary/個人app/個人資料夾/組織運作資料庫.md` (若無此檔案則自動建立)

---

## ⚙️ 觸發時機與執行標準 (Trigger & SOP)

**當使用者（Gary）要求：**
「整理今日工作日誌」、「產出 WorkLog」、「幫我寫今日小記」時，Agent 在生成完日誌內容後，必須**主動執行以下附加步驟**。

### 步驟一：分析與提煉 (Extraction)
掃描當日的工作紀錄與對話，尋找以下三大類經驗值：
1. **業務決策 (Business Decisions)**：
   * 做了什麼關鍵決定？
   * 為什麼這樣決定（背後的權衡與考量是什麼）？
2. **管理經驗與踩坑紀錄 (Management Experiences)**：
   * 遇到了什麼阻力或跨部門拉扯？
   * 最終用哪種方式（流程改變、溝通技巧、底層邏輯）解決的？
3. **系統與架構洞察 (System Insights)**：
   * 發現了什麼系統漏洞或流程瓶頸？
   * 提出了什麼自動化或防呆機制構想？

### 步驟二：抽象化與底層邏輯對應 (Abstraction)
將提煉出的具體事件「降維/抽象化」，並嘗試對應到已知的 Skill 框架（如《底層邏輯》、《劉潤思維》、《核心主管風格》），寫成通用的「指導原則」。

### 步驟三：寫入資料庫 (Append to Database)
使用 `run_command` 或檔案編輯工具，將提煉出的經驗值以特定的格式「附加 (Append)」到 `組織運作資料庫.md` 的最上方或對應的分類下。

---

## 📝 寫入格式範例 (Template)

每次寫入資料時，請遵循以下 Markdown 格式：

```markdown
### 🗓️ [YYYY-MM-DD] [一句話總結經驗核心]
* **觸發情境**：(簡短敘述當天遇到的問題或決策情境)
* **底層邏輯分析**：(套用系統思維、分錢邏輯、商人的對錯觀等進行分析)
* **決策與解法**：(具體採取的行動或建立的流程)
* **未來可複用性**：(未來若遇到類似狀況，應該直接採取的 SOP 或原則)
```

---

## 🤖 給 Agent 的重要提醒
* **不要只是複製貼上**：日誌是「流水帳」，資料庫是「智慧結晶」。你必須扮演「系統建築師」，把流水帳轉化為「演算法/SOP」。
* **主動回報**：每次完成日誌產出後，在回覆的最後加上一段：「✅ *已將今日關於 [某某決策] 的經驗提煉並同步至『組織運作資料庫』中。*」，讓 Gary 知道系統正在為他累積資產。
