## 格式規範

### 字型與基本設定
- **字型**：標楷體（所有文字，含標題、表格、東亞字元都必須設定 `w:eastAsia`）
- **內文字型大小**：12pt
- **行距**：1.15 倍
- **頁面邊距**：上下左右各 2.54cm

### 封面頁
| 元素 | 大小 | 格式 | 對齊 |
|------|------|------|------|
| 機構名稱（兩行） | 16pt | 一般 | 置中 |
| 計畫名稱 | 22pt | **粗體** | 置中 |
| 計畫資訊（執行人等） | 14pt | Label 粗體、Value 一般 | 置中 |
| 日期 | 16pt | 一般 | 置中 |

**封面排版要點**：
- **不加頂部空行**：機構名稱直接從頁面頂端開始，不需要在上方加空行
- **日期放首頁 Footer（頁尾）**：使用 Word「不同首頁」功能，將日期放在首頁的 footer 中，確保無論上方段落如何調整，日期永遠固定在頁面最底部。不要用空行來控制日期位置，那樣不穩定。

```python
# 啟用「不同首頁」，日期放 footer
section.different_first_page_header_footer = True
footer = section.first_page_footer
fp = footer.paragraphs[0]
# 設定置中、16pt 標楷體
```

### 內文頁標題
- **不重複機構名稱**，僅保留計畫名稱（16pt 粗體 置中）作為內文頁的標題
- 封面已有完整機構名稱，內文頁不需再重複

### 大標章節（壹、貳、參...）
- 15pt 粗體
- space_before = 18pt, space_after = 6pt

### 次標章節（一、二、三...）
- 13pt 粗體
- left_indent = 0.8cm
- space_before = 12pt, space_after = 6pt

### 列點（一）（二）（三）
- 12pt
- left_indent = 2.4cm, hanging_indent = 1.6cm
- 標題部分粗體，冒號後的說明內容為一般字重

### 內文段落
- 12pt
- 首行縮排 1cm（`first_line_indent`）
- space_after = 6pt

### 大綱編號層級對照（嚴格遵守！）

| 層級 | 編號格式 | 範例 |
|------|----------|------|
| 第1層 | 壹、貳、參… | 壹、前言 |
| 第2層 | 一、二、三… | 一、目標 |
| 第3層 | （一）（二）（三）… | （一）擴充設備 |
| 第4層 | 1. 2. 3. | 1. 電腦 |
| 第5層 | (1) (2) (3) | (1) 規格說明 |

> **⚠️ 常見錯誤**：在「壹」級標題下直接使用（一）（二）（三）——這是跳層。應先有「一、二、三、」再往下展開到（一）（二）（三）。

### 預期效益專用格式
- **必須使用條列式**，不要寫成一整段話
- 層級：在「陸、預期效益」下使用「一、二、三、」（第2層）
- 每一條格式：`一、【標題粗體】：【說明內容】`
- left_indent = 1.6cm, hanging_indent = 0.8cm

## 經費概算表格

### 表格結構
- 使用 `Table Grid` 表格樣式
- 包含 7 欄：項次、項目及型號、單位、數量、單價、複價、備註/參考連結
- 最後一行為「總計」列，前 6 欄合併靠右顯示「總計」文字

### 欄位對齊
| 欄位 | 對齊 |
|------|------|
| 項次、單位、數量 | 置中 |
| 單價、複價 | 靠右 |
| 項目名稱 | 靠左 |
| 備註/參考連結 | 靠左 |

### 表格字型：10pt 標楷體

### 表格上方備註
- 「單位：新台幣（元）皆已含稅」→ **靠右對齊**

### 表格下方備註
- 僅保留必要的註解（如素食說明）
- **不需要**加「採購單價參考供應商報價」等冗餘說明

### 參考網址處理
- 若有採購物品的參考連結，加在表格的「備註/參考連結」欄
- **務必精簡 URL**：移除所有追蹤參數（`utm_*`, `gclid`, `gbraid`, `gad_*`, `gads_*`, `osm` 等），僅保留商品識別所需的最小參數
- 連結以**超連結**方式嵌入（可點擊），顯示文字用平台名稱（如「momo 購物網」「蝦皮購物」「特力屋」「PChome 24h」）

#### URL 精簡範例
```
原始：https://www.momoshop.com.tw/goods/GoodsDetail.jsp?i_code=13444243&osm=Ad07&utm_source=googleshop&gclid=...
```

## 前言固定段落

前兩段為機構沿革，除非有重大變更否則保持不變：

**第1段**：創辦緣起（八七水災 → 慈光育幼院 → 基金會 → 慈馨兒少之家）

## 文案撰寫原則

### 物品命名原則
- **使用用途導向名稱**，不要使用技術規格名稱
- 範例：「兒少學習用電腦」✅ 而非「電腦處理器」❌ 或「電腦主機」❌
- 命名一旦決定，**全文統一使用同一名稱**，包括：前言、目標、實施內容、經費表格、預期效益

### 功能取向用語
- **描述物品效益時，聚焦在實際功能**，不要過度延伸到抽象目標
- ✅ 「提供收納孩子生活用品之容器，有效收納及管理生活物資」
- ❌ 「培養孩子良好的整理收納習慣，提升生活自理能力」
- 簡單來說：**寫物品能做什麼，不要寫希望孩子變成什麼**

### 前後一致性檢查（必做！）
產出完畢後，必須進行以下一致性核對：
1. **物品名稱一致**：前言、目標、實施內容、經費表格、預期效益中的物品名稱是否完全相同？
2. **論述邏輯一致**：目標的描述 → 實施內容的展開 → 預期效益的呼應，三者邏輯線是否一貫？
3. **語氣風格一致**：如果某處修正了用語（例如不寫「培養習慣」），所有相關段落都必須連動修正
4. **數字一致**：物品數量、單價、總計金額在實施內容與經費表格中是否吻合？

每個 `add_run` 都必須設定此屬性，否則標楷體不會套用到中文字元。

### 超連結加入方式
```python
def add_hyperlink(para, url, display_text, font_size=11, font_name='標楷體'):
    part = para.part
    r_id = part.relate_to(
        url,
        'http://schemas.openxmlformats.org/officeDocument/2006/relationships/hyperlink',
        is_external=True
    )
    hyperlink = parse_xml(
        f'<w:hyperlink r:id="{r_id}" xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main"'
        f' xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships">'
        f'<w:r><w:rPr><w:rStyle w:val="Hyperlink"/>'
        f'<w:rFonts w:eastAsia="{font_name}"/>'
        f'<w:sz w:val="{font_size * 2}"/>'
        f'<w:color w:val="0563C1"/><w:u w:val="single"/>'
        f'</w:rPr><w:t>{display_text}</w:t></w:r></w:hyperlink>'
    )
    para._element.append(hyperlink)
```

## 踩坑記錄

| 問題 | 原因 | 解法 |
|------|------|------|
| 內文頁重複出現機構名稱 | 封面與內文都放了全名 | 內文頁只放計畫名稱，不重複機構名稱 |
| 「單位：新台幣…」沒靠右 | 預設靠左 | 設定 `alignment=WD_ALIGN_PARAGRAPH.RIGHT` |
| 表格下方多餘的「註2」 | 寫了供應商報價參考 | 刪除不必要的備註，保持精簡 |
| 收容人數過時 | 使用固定數字 | 每次產出前確認最新人數（目前 13 位） |
| 參考連結太長、含追蹤碼 | 直接貼上原始 URL | 精簡 URL，移除追蹤參數，用平台名稱作為超連結顯示文字 |
| 東亞字型沒生效 | 只設定 `font.name` | 必須額外設定 `rFonts.set(qn('w:eastAsia'), '標楷體')` |
| 封面日期用空行控制位置 | 內容增減時日期會跑版或溢出到第2頁 | 改用 Footer（首頁頁尾）固定日期位置 |
| 預期效益寫成一大段 | 不易閱讀 | 改為條列式（一、二、三、），每條一個主題 |
| 大綱層級跳層 | 壹級下直接用（一）（二） | 嚴格遵照層級：壹→一→（一）→1.→(1) |
| 物品名稱不統一 | 同一物品在不同段落用不同名稱 | 全文統一名稱，修改一處就必須全文連動 |
| 收納用品寫成「培養習慣」 | 過度延伸到教育目標 | 聚焦物品的實際功能（收納、管理物資） |
