# GTD 任務處理準則

## 術語簡稱
- **1m** = Notion「近一個月的任務規劃」(Page ID: `30d1fbf9-30df-81bb-9dc5-d6a2418dc5f3`)
- **task** = Notion「Tasks (每日行動)」資料庫

## 新增任務標準流程

### Step 1：Notion 同步
1. 在 **task** 裡新增一筆任務（或專案頁面 + 子任務）
2. 該任務連動更新到 **1m** 裡面
3. **⚠️ Sync Block 規則**：
   - task 專案（如 p1）底下的子任務，必須用 Notion **Sync Block** 同步到 1m 對應段落
   - 一次性新增多個子任務時，用**一個 sync block 框住全部**，不需每個 block 各設一個
   - 後來追加的子任務也必須同步加到 1m，保持兩邊一致

### Step 2：Things 3 同步（由 @gaa_macbook_air_bot 執行）
1. **開始日期**：若 Gary 有指定日期就用；若沒有，Air 根據掌握的所有行事曆資訊，先安排一個時間，並問 Gary 確認。
2. **Tag（標籤）**：依 GTD Context 標籤分類（@電腦前、@電話、@外出、@人名、@會議名稱等）
3. **專案歸屬**：
   - 若屬於某個專案 → 放到該專案底下
   - 若不屬於任何專案 → 放到「📦行政/雜事」
   - **⚠️ 大型任務（多步驟、多人、有 deadline）必須用「專案 (Project)」建構**，子任務掛在專案底下，不可散列
4. **專案判斷**：問 Gary 這是否為「專案」（目標/大項目）
   - **是專案** → 執行項目拆解流程：
     a. 參考 Notion「小強升職記行動方案」(Page ID: `06bf276d-4274-4f4d-9ec3-6b044ba542a7`) 的項目規劃部分
     b. 依 GTD 原則拆解為可執行的下一步行動
     c. 每個子任務都要有開始日期和標籤
   - **不是專案** → 作為單一行動處理
5. 完成後在回覆末尾加上「things3」標記

### Step 3：監督（由 @gaa1243_bot 執行）
- 檢查 Air 的回覆有沒有「things3」標記
- 若發現任務遺漏，提醒 Air 補入
- 一併檢查：開始日期、標籤、專案歸屬是否完整

## GTD 決策流程（收到任務時先走這個）

```
收到任務
  ├─ 這是什麼？可以執行嗎？
  │   ├─ 不可執行 → 丟掉 / 放參考資料 / 放「將來也許」
  │   └─ 可執行 ↓
  ├─ 2 分鐘內能完成嗎？
  │   ├─ 是 → 立刻做
  │   └─ 否 ↓
  ├─ 是我該做的嗎？
  │   ├─ 不是 → 委託他人（Q3）
  │   └─ 是 ↓
  ├─ 需要多個步驟嗎？
  │   ├─ 是 → 這是「專案」，需拆解
  │   └─ 否 → 單一行動
  └─ 有特定時間嗎？
      ├─ 是 → 放行事曆
      └─ 否 → 放 Next Actions（加 Context 標籤）
```

## 四象限對應
- **Q1（重要且緊急）**：Things 3 標籤 `!1 立刻去做 ★★★`，排程 Today
- **Q2（重要不緊急）**：Things 3 標籤 `!2 計畫去做 ★★`，排程 Anytime
- **Q3（委託他人）**：Things 3 標籤 `!3 授權去做 ★`，排程 Anytime
- **Q4（不重要不緊急）**：不記入，或放「將來也許」
