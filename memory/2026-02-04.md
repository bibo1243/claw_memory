## 群組 Session 無法讀取 Sheets 問題 ✅

### 問題
群組 session 查假表時報 403 錯誤，說 `deleted_client`

### 原因
1. Gateway config 裡的環境變數已更新為新憑證
2. 但群組 session 啟動時**快取了舊的環境變數**
3. 重啟 Gateway 後，環境變數**不一定立即同步**到所有 session

### 解決方法
直接把正確的憑證傳給群組 session，讓它用硬編碼方式查詢，而不是依賴 `os.getenv()`

### 教訓
1. **環境變數更新後，不同 session 可能有不同的快取狀態**
2. **跨 session 同步設定時，不能假設環境變數已更新**
3. **敏感憑證可以透過 sessions_send 傳遞給其他 session 使用**
